# 🤖 智能体标准模板

> 创建新智能体的标准化模板，确保一致性和高质量

---

## 📋 Front Matter 规范

每个智能体文件必须以 YAML Front Matter 开头：

```yaml
---
name: 智能体名称
description: 一句话描述智能体的核心能力（30字以内）
color: "#十六进制色值"
difficulty: 初级|中级|高级
tags: [标签1, 标签2, 标签3]
version: "1.0.0"
author: "作者名"
---
```

### 字段说明

| 字段 | 必填 | 说明 | 示例 |
|------|:----:|------|------|
| `name` | ✅ | 智能体名称，简洁有力 | 前端开发专家 |
| `description` | ✅ | 一句话描述，突出核心价值 | React/Vue/Angular 界面实现专家 |
| `color` | ✅ | 主题色，用于标识 | "#3178c6" |
| `difficulty` | ✅ | 难度等级 | 初级/中级/高级 |
| `tags` | ✅ | 标签数组，便于分类 | ["Web", "React", "前端"] |
| `version` | ❌ | 版本号 | "1.0.0" |
| `author` | ❌ | 作者名 | "张三" |

### 难度标识

- 🟢 **初级**：通用型，易于使用，适合新手
- 🟡 **中级**：需要一定领域知识，适合有基础的用户
- 🔴 **高级**：专业深度，需要深入理解领域

## ✅ 质量检查清单

在提交新智能体前，请确认以下项目：

### Front Matter
- [ ] 包含所有必填字段
- [ ] `name` 简洁有力（不超过10个字）
- [ ] `description` 在30字以内
- [ ] `color` 使用有效的十六进制色值
- [ ] `difficulty` 准确反映复杂度
- [ ] `tags` 包含 2-5 个相关标签

### 内容质量
- [ ] 角色定义清晰具体
- [ ] 有独特的性格特征（非通用助手）
- [ ] 核心使命明确，交付物具体

- [ ] 经过真实场景测试

## 🎨 风格指南

### 命名规范

- **智能体名称**：领域 + 角色，如"前端开发专家"
- **文件命名**：`领域缩写-角色名称.md`，如`engineering-frontend-developer.md`
- **颜色选择**：参考该领域常用品牌色或工具主题色

### 写作风格

1. **具体明确**
   - ❌ 写："我会帮你优化代码"
   - ✅ 写："将页面加载速度降低 60%，Core Web Vitals 全部达标"

2. **落地务实**
   - ❌ 写："提供组件建议"
   - ✅ 写："用 TypeScript 编写 React 组件，包含 Props 类型定义和单元测试"

3. **个性鲜明**
   - ❌ 写："我是一个有帮助的 AI 助手"
   - ✅ 写："对细节有执着追求的像素级还原专家"

4. **实用可用**
   - 提供可直接运行的代码
   - 给出具体的命令和参数
   - 包含真实场景的示例
