## 报告安全问题

我们非常重视安全问题。如果您发现了安全漏洞，请按照以下流程报告：

### 🔒 负责任的披露

**请不要**通过公开的 GitHub Issue 报告安全漏洞。

相反，请通过以下方式私下报告：

1. **发送邮件至**: [待填写安全联系邮箱]
2. **邮件主题**: `[Security] 漏洞简要描述`
3. **包含内容**:
   - 漏洞类型
   - 受影响的文件/功能
   - 重现步骤
   - 可能的影响
   - 建议的修复方案（如有）

### 响应时间线

| 阶段 | 时间 |
|------|------|
| 初步确认 | 48 小时内 |
| 评估严重性 | 5 个工作日内 |
| 修复计划 | 10 个工作日内 |
| 修复发布 | 视严重性而定 |

## 安全最佳实践

### 对于用户

1. **API 密钥保护**
   - 永远不要将 API 密钥硬编码在代码中
   - 使用环境变量或密钥管理服务
   - 定期轮换 API 密钥

2. **脚本安全**
   - 审查脚本内容后再执行
   - 使用只读权限运行安装脚本
   - 检查脚本来源

3. **数据隐私**
   - 不要在智能体文件中包含敏感信息
   - 避免在公共仓库中提交个人数据

### 对于贡献者

1. **代码审查**
   - 所有脚本必须包含错误处理
   - 避免使用 `eval` 或类似危险函数
   - 验证所有用户输入

2. **依赖管理**
   - 使用固定版本号
   - 定期检查依赖的安全更新
   - 最小化依赖数量
